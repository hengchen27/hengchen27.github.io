This piece was created for an assignment which is basically a JavaScript Game using CreateJS for Special Topic I Class. This game has the following features: (1) different game levels with different patterns of bricks to break (2) different types of bricks (i.e. a different colour bricks that takes more than one hit to break and change colour to show breaking progress) (3) special bricks that drop power ups that your paddle can catch for upgrades and one of these power-ups must be a second ball (4) a timer for each level so bricks have to be broken within the time frame (5) High score functionality stored in local storage (6) Animation with SpriteSheet, sound files for game events, both keyboard and mouse controls, and show progress bar when loading the game.
The screenshot on the right shows the starting screen after choosing the level 4.

Check out finished Break Out Game
The highlight of this JavaScript game application is to use CreateJS, which is to create animations (SpriteSheet) and interactive content（ for example SoundJS and PreloadJS） on our HTML5 pages in combination with basics of HTML5 Canvas and User Input by Keyboard and Mouse. Besides. As for game design itself, it is a dramatically beefed up version of simple break out game that was learned and built through a tutorial.

Following the requirements(features) of this assignment, I started with designing the game levels, brick patterns and brick colors logic as well as a countdown timer. After that, I added animations by using Sprite Sheet for showing drop-drown items when hitting special bricks with its logic. In addition to that, I added several sound files and a game loading progress bar with CreateJS. The whole process is done step by step by following the requirements of an advanced version of breakout game.

  • Skills: HTML/CSS/JavaScript/CreateJS/SpriteSheets/Bootstrap/Jquery
  • Tools: TexturePacker(sprite sheet generation tool)/ Pixilart(free online pixel drawing tool), Visual Studio Code(code editor)

By building this application, I have picked up and strengthened the skills gained in year 1 like HTML/CSS/JavaScript and learned how to use canvas/ add animations, and incorporate other user input by using keyboard and mouse controls, most importantly, I got to understand how to make use of CreateJS for animations and interactive contents. The biggest challenges are preparing Sprite Sheet and designing the animation parts for the game.

This application was created for one of the assignments in Client-side Programming in Year 1, which is basically a single HTML page library system that will load a list of available books from the server. Users are able to download books from the library and store them locally for reading anywhere. As for the layout of the book reader, there are two views-the first view gives the user a list of books that are available in the system, and the second is the reading view containing a table of contents and a reading pane.
The screenshot on the right shows the second view of the E-reader.

Check out finished E-Reader

The highlight of this assignment is about “caching”, and in this case we are storing 3 main types of data: the reader (our webpage), the manifest (the list of books), and the books themselves. Other specific goals are displayed as below:

  • Manifest is refreshed periodically
  • Manifest is downloaded
  • Book List is stored in database
  • Book List shows which books have been downloaded and which are available for download
  • Switch between Views
  • Selection of book transitions to reader view
  • Books are stored in database and retrieved from it
  • Table of Contents is generated
  • Button to delete all stored data is available.

The process started by identifying the basic folder structure, the reader, the manifest and books folder. I also created a readerHelper class for methods dealing with things like updating the book list on the page, downloading books, removing books, creating book list and updating book content etc. There is also a reader.js for creating an object of the readerHelper class and defining book class and database object to store all the books as well as a timer for continuously pulling data.

  • Skills: HTML/CSS/Vanilla JavaScript/bootstrap
  • Tools: WebStorm/books resources

By building this E-reader application, I have gained a better understanding of several topics including HTML Web storage, Javascript class, manifest, and how to update caches periodically.

The screenshot on the right shows the page of course list when app starts up.

Check out my work on Firebase

The application will display a list of Courses when the application starts up. These courses should be accessed via your live NSCC Course Map API.

When a user clicks on or taps a course, it should bring them to another page which will display the details for that course, including the prerequisites and the isprerequisitefor collections.

The app should employ some caching strategy described as below:

  • The app should cache the app shell and use the cached version until the cache version is updated in the service worker file.
  • The app should cache a list of courses upon successful retrieval of the courses data from the API.
  • The app should cache any course details for any selected course from the main list.

Deploy the app to some hosting platform to be accessible like Firebase.

The biggest part of the work is to successfully call the course map api (I added a new CORS policy in the program.cs of my api project to make it work). This piece was created for an assignment for my Full stack Programming class. This application allows user to enter keyword in search of tv shows (api data from TV Maze), and the search result will show 10 records that matches the user’s search. When the user clicks on any tv show from the list, more details of this tv show will be displayed on the right of the panel, and there is a cast list button which will display the cast information when clicked. This will be kept open when the user switches between different tv shows until he/she tries a new search. The screenshot on the right shows the screen when user starts a research and clicks on one of the tv shows.

View my code on Github Repo
  • The application will need to maintain the state necessary for the correct functioning of the application
  • The application will need to make at least two separate calls to the TV Maze API to retrieve all of the required data.
  • It allows the end-user to enter a search term in a text box and trigger a search either by clicking an accompanying button or by leveraging the change event of the text box >
  • It allows the end-user to click on a single search result to trigger the available details of the show to appear in the ShowDisplay component
  • The tv show details section displays the image for the selected show when available. If an image is not available, a generic image informing the end-user that an image is not available should be displayed.
  • It initially displays a button that prompts the user to click in order to retrieve cast data from the API about the currently selected show. Once clicked, the component should then display either a list if cast results, or if no results are returned, a message stating so. As for Cast list data, once loaded for a selected show, should not have to be reloaded until such a time as a new search is performed and a new set of search results is returned.

The React Application was completed by taking many steps, and the first and most important step was to build all the required components and determine the parent-child relationships of them. After that, I followed the instructions for each component to display the required data called from TV Maze API .

  • Skills: ReactJS/Javascript/html/CSS/W3.css/
  • Tools: Visual Studio Code/TVMaze API (data resource)

This is the first React Application I built after learning ReactJS. It has reflected all the key concepts in React like props, state, JSX, components and lifecycle methods. It has used the data from TVMaze API in an informative and clear way that will be a very useful tv show search tool for users.
